## 8. Análise detalhada do ToolBox > Editor na página Presentation (2026-05-26)

### Cenário Atual

A análise é focada na página "presentation", especificamente no componente ToolBox no item Editor. As observações são baseadas no cenário atual apresentado nas imagens anexadas.

#### Pontos Observados

- O texto "ACTIVE LAYOUT" aparece acima da seleção de layout (Expanded layout ou Default layout).
- O nome do layout (Expanded layout ou Default layout) aparece dentro do card branco, junto com as demais configurações.
- O fundo do card de "active layout" é branco, destacando as opções.
- Existem controles para "Font", "Columns", "Marks Visible" e "Detected Blocks".
- O botão "Show Marks" dentro do Editor não funciona, mas o mesmo botão dentro de ToolBox > Layout está funcional.
- O "Mark Editor" não exibe as marks, impossibilitando a edição de colunas e nomes.

### Pontos de Melhoria e Mudanças Desejadas

1. **Escolha de Layout e Configurações Persistentes**

- O usuário deve poder escolher entre layouts (default ou expanded).
- Cada layout deve ter suas próprias configurações de editor, persistidas separadamente.

2. **Ajustes Visuais e de Interface**

- Remover o texto "active layout".
- O nome do layout (Expanded layout / Default layout) deve ser exibido em uma linha solo acima do ícone "LIVE", com cor dourada e formatação igual ao texto "VOICING" da página Chord Library.
- Remover o fundo branco do card de "active layout".
- O seletor de tamanho de fonte deve ser igual ao utilizado em ToolBox > Font Size.
- Remover os controles de "Columns", "Marks Visible" e "Detected Blocks" do Editor.

3. **Funcionalidade de Marks**

- O botão "Show Marks" dentro do Editor não funciona; apenas o de ToolBox > Layout está funcional.
- O "Mark Editor" não exibe as marks, impossibilitando a edição.
- Sugestão: Adicionar um editor de marks integrado à função de editar cifra, com um botão que ativa uma interface para editar posição e nome das marks existentes.

### Resumo

O cenário atual apresenta limitações de usabilidade e inconsistências visuais. As mudanças propostas visam tornar a experiência mais intuitiva, centralizando a configuração por layout, melhorando a interface e tornando a edição de marks funcional e acessível.
